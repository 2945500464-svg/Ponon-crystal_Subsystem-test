# 数据结构说明

本程序以 `Raw_data` 作为唯一数据入口。每个日期或批次建立一个子文件夹，文件夹名用于匹配 `数据格式.xlsx` 中的结构配置。

## 1. 目录约定

```text
Raw_data/
├─ 数据格式.xlsx
├─ 5.8/
├─ 5.10/
├─ 5.11/
├─ 5.12/
├─ 5.22/
├─ 5.23/
└─ 5.31/
```

`数据格式.xlsx` 放在 `Raw_data` 根目录下。程序会优先读取该表；没有表格或表格缺少某个日期时，使用内置兜底规则。

## 2. 数据格式表

建议 `数据格式.xlsx` 至少包含以下字段：

| 字段 | 含义 | 示例 |
|---|---|---|
| 文件夹命名 | 日期或批次文件夹名 | `5.8`、`5.12` |
| 激振器位置 | 输入基准选择依据 | 左边、右边 |
| 传感器数量 | 通道结构识别依据 | 8、9、10 |

输入基准规则：

| 激振器位置 | 默认输入基准 |
|---|---|
| 左边 | `Data 第 1 行` |
| 右边 | `Data 第 2 行` |

GUI 中保留输入基准手动覆盖选项，但默认建议使用自动规则。

## 3. `.mat` 文件要求

每个 `.mat` 文件代表一个工况。工况名称默认取文件名去除扩展名后的结果。

```text
shdf
├─ Absc1Data   时间数据，通常为 1 × N
└─ Data        通道数据，通常为 M × N
```

程序只按行号解释通道，不根据 `.mat` 内部通道名称或单位元数据自动判断输入输出。

## 4. 内置通道结构

### 4.1 old8

旧 8 通道结构：

| Data 行号 | 默认名称 |
|---|---|
| 1 | 左输入 |
| 2 | 右输入 |
| 3 | 左前 |
| 4 | 左后 |
| 5 | 左动力吸振器 |
| 6 | 右前 |
| 7 | 右后 |
| 8 | 右动力吸振器 |

统计测点：

- 四点：左前、左后、右前、右后。
- 六点：左前、左后、左动力吸振器、右前、右后、右动力吸振器。

### 4.2 force9

9 通道结构，在旧结构中加入力传感器：

| Data 行号 | 默认名称 |
|---|---|
| 1 | 左输入 |
| 2 | 右输入 |
| 3 | 力传感器 |
| 4 | 左前 |
| 5 | 左后 |
| 6 | 左动力吸振器 |
| 7 | 右前 |
| 8 | 右后 |
| 9 | 右动力吸振器 |

统计测点：

- 四点：左前、左后、右前、右后。
- 热力图、总振级和平均减振率默认使用四点，除非表格结构明确含左中/右中。

### 4.3 new10

10 通道结构，在力传感器下方加入激振器加速度：

| Data 行号 | 默认名称 |
|---|---|
| 1 | 左输入 |
| 2 | 右输入 |
| 3 | 力传感器 |
| 4 | 激振器加速度 |
| 5 | 左前 |
| 6 | 左后 |
| 7 | 左中 |
| 8 | 右前 |
| 9 | 右后 |
| 10 | 右中 |

统计测点：

- 四点：左前、左后、右前、右后。
- 六点：左前、左后、左中、右前、右后、右中。
- 热力图、总振级和平均减振率会自动使用六点。

## 5. 麦克风数据结构

麦克风数据固定放在 `Raw_data/5.31`，Data 只有三行：

| Data 行号 | 麦克风位置 | 单位 |
|---|---|---|
| 1 | 主驾驶 | Pa |
| 2 | 中排 | Pa |
| 3 | 后排 | Pa |

麦克风流程独立于副车架振动流程，不使用输入基准、输出测点或传递率通道结构。
